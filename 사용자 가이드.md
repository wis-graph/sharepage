# 🚀 SharePage 사용자 가이드

SharePage는 옵시디언(Obsidian) 문서를 가장 아름답고 스마트하게 공유할 수 있는 플랫폼입니다. 이 가이드는 문서를 업로드하고 멋진 미리보기를 만드는 방법을 설명합니다.

## 1. 문서 게시 방법 (GitHub Pages)

SharePage는 **GitHub Actions**를 통해 배포가 자동화되어 있습니다.

- **업로드**: 작성한 `.md` 파일을 레포지토리의 `gh-pages` 브랜치에 업로드(Push)하기만 하면 됩니다.
- **자동화**: 파일을 올리면 깃허브 서버에서 **미리보기용 정적 HTML 생성**부터 **배포**까지 약 1분 안에 자동으로 완료합니다.
- **확인**: 배포가 완료되면 `https://<사용자명>.github.io/<저장소명>/` 주소에서 바로 확인할 수 있습니다.

## 2. 문서 작성 및 메타데이터 (YAML)

문서 최상단에 YAML Frontmatter를 추가하여 페이지 설정과 미리보기를 제어할 수 있습니다.

```yaml
---
title: 문서 제목 (미출력 시 파일명 사용)
description: 카카오톡/슬랙 미리보기에 표시될 요약 문장
thumbnail: 썸네일 파일명 또는 URL
tags:
  - "#태그1"
  - "#태그2"
---
```

## 3. 오픈그래프(OG) 및 미리보기 최적화

카카오톡이나 슬랙에 링크를 공유했을 때 나타나는 정보는 다음과 같이 결정됩니다.

- **제목**: YAML의 `title` 값을 사용합니다.
- **설명**: YAML의 `description` 값을 사용합니다. (없을 경우 본문의 앞부분을 자동으로 요약합니다.)
- **썸네일 이미지**: 핵심 우선순위는 다음과 같습니다.
    1.  `thumbnail` 필드에 지정된 이미지
    2.  본문에 삽입된 첫 번째 유튜브 영상의 썸네일 (자동 추출)
    3.  본문에 삽입된 첫 번째 이미지

## 4. 유튜브(YouTube) 처리 방식

SharePage는 유튜브 링크를 지능적으로 처리합니다.

- **자동 임베드**: 본문에 유튜브 링크(`https://www.youtube.com/watch?v=...`)를 적으면 자동으로 **반응형 비디오 플레이어**로 변환됩니다.
- **썸네일 자동 생성**: 문서의 `thumbnail`을 따로 지정하지 않아도, 영상 링크만 있으면 해당 영상의 가장 고화질 썸네일을 가져와 미리보기 이미지로 사용합니다.

## 5. 이미지 및 첨부파일

- **로컬 이미지**: 이미지 파일명을 `_image_파일명` 형태로 저장소에 함께 올리면 옵시디언 방식(`![[파일명]]`)으로 바로 사용할 수 있습니다.
- **외부 이미지**: 웹상에 이미 있는 이미지 주소(`https://...`)도 그대로 사용 가능합니다.

---

### 💡 팁
- **정적 HTML 생성**: SharePage는 검색 엔진과 SNS 봇을 위해 각 문서마다 정적 HTML 껍데기를 생성하므로, SEO에 매우 유리합니다.
- **다크 모드**: 우측 상단의 테마 스위치를 통해 독자가 원하는 테마로 문서를 읽을 수 있습니다.
